# 【UI/実装仕様書】経済・名声システムの拡張および移動エンカウント仕様 (spec_v16)

## 概要
ゲーム内経済のインフレ抑制（ゴールドシンクの強化）、名声（Reputation）のメリット・デメリットの明確化、および「移動時」のゲーム体験向上のための拡張仕様です。

## 1. 移動中のエンカウントバトル仕様
ワールドマップでの拠点間移動（`POST /api/move`）における、道中バトルの発生。

### 1.1 通常のランダムエンカウント
- **発生確率**: `RANDOM_ENCOUNTER_RATE` (初期値: 20%)
- **挙動**: 移動API実行時、判定に成功すると目的地到着前に「移動中バトルノード（`require_battle`）」へ遷移。
- **敵の選出**: 出発地または目的地の適正レベル範囲のエネミーからランダム選出。
- **勝敗判定**:
  - **勝利**: その後目的地へ到着。
  - **敗北/退却**: Vitalityを失い、出発地の拠点へ強制送還（移動失敗）。

### 1.2 賞金稼ぎ（Bounty Hunters）との確定エンカウント
- **条件**: **出発地**での名声（Reputation）が `BOUNTY_HUNTER_THRESHOLD` (初期値: -100) 以下。
- **発生確率**: **100%（確定）**。通常エンカウントより優先。
- **敗北ペナルティ**: 死亡（Vitality 0）は免除されるが、所持金の `BOUNTY_PENALTY_RATE` (初期値: 50%) を没収され、出発地の拠点へ強制送還。

## 2. 首都への入場制限
王都レガリア、黄金都市イスハーク、神都「出雲」、天極城「龍京」への立ち入り制限。

### 2.1 通行許可証 (Pass)
- **アイテム定義**: `key_item` として定義。
- **価格**: `PASS_PRICE` (初期値: 20,000G)。首都以外の道具屋で販売。
- **有効期限**: `user_profiles.pass_expires_at` に「現在の `age_days` + `PASS_DURATION_DAYS` (初期値: 365日)」を記録。

### 2.2 賄賂 (Bribe)
- **条件**: 名声が 0 未満の場合のみ選択可能。
- **費用**: `BRIBE_COST` (初期値: 10,000G)。
- **挙動**: 有効な許可証がない場合でも、賄賂を支払うことで強行突破（入城）が可能。

## 3. 名声（Reputation）システムの拡張

### 3.1 クエスト失敗ペナルティ
- **仕様**: バトル敗北やリタイアによるクエスト失敗時、その拠点での名声を `QUEST_FAIL_REP_PENALTY` (初期値: -5 〜 -10) 減少させる。

### 3.2 名声の自然減少 (Decay)
- **仕様**: 日次バッチにおいて、名声が `REP_DECAY_THRESHOLD` (初期値: 100) を超えている場合、毎日 `REP_DECAY_AMOUNT` (初期値: -5) ずつ自然減少。

### 3.3 クエスト受領条件の拡張
- **仕様**: `scenarios` テーブルに `max_reputation` (受注可能な名声の上限) を追加。
- **活用**: 「悪党限定クエスト」や「英雄限定クエスト」のマスタ設定。

## 4. 追加のゴールドシンク (Gold Sinks)

### 4.1 名声ロンダリング（身分洗浄）
- **場所**: 崩壊拠点に出現する「闇市」。
- **費用**: `LAUNDERING_COST` (初期値: 100,000G)。
- **効果**: 任意の拠点（または全拠点）の名声を 0 にリセット。

### 4.2 英霊（残影）の摩耗
- **仕様**: **宿屋で休んでも、傭兵（NPC/残影）のHPは回復しない。**
- **回復手段**: バトル中の回復スキル等に限定。
- **目的**: 高額な再契約サイクルの創出。

### 4.3 祈り（Prayer）の強化
- **首都ボーナス**: 首都での祈りによる属性影響力を `CAPITAL_PRAYER_MULTIPLIER` (初期値: 2.0倍) に設定。
- **個人バフ (Blessing)**: 祈り実行時、次のバトル終了時まで以下のバフを付与。
  - `PRAYER_BUFF_HP_PCT` (初期値: 10%)
  - `PRAYER_BUFF_AP` (初期値: 1)
