# 【UI/実装仕様書】経済・名声システムの拡張および移動エンカウント仕様 (spec_v16)

## 更新履歴
- **v16.1** (2026-04-04): エンカウント敗北時送還先を出発地に修正、ロンダリング場所制限追加

## 概要
ゲーム内経済のインフレ抑制（ゴールドシンクの強化）、名声（Reputation）のメリット・デメリットの明確化、および「移動時」のゲーム体験向上のための拡張仕様です。

## 1. 移動中のエンカウントバトル仕様
ワールドマップでの拠点間移動（`POST /api/move`）における、道中バトルの発生。

### 1.1 通常のランダムエンカウント
- **発生確率**: `RANDOM_ENCOUNTER_RATE` (初期値: 20%)
- **挙動**: 移動API実行時、判定に成功すると目的地到着前に「移動中バトルノード（`require_battle`）」へ遷移。
- **敵の選出**: 出発地の `location_encounters` テーブルから重み付き選出（`encounter_type = 'random'`）。
- **勝敗判定**:
  - **勝利**: その後目的地へ到着。
  - **敗北/退却**: Vitalityを失い、**ワールドマップ画面に出発地を選択した状態で強制送還**（拠点には直接入らない）。
  - `POST /api/move/encounter-result` の `lose` 時に `current_location_id = origin_location_id`、レスポンスに `redirect_to_map: true` を含む。

### 1.2 賞金稼ぎ（Bounty Hunters）との確定エンカウント
- **条件**: **出発地**での名声（Reputation）が `BOUNTY_HUNTER_THRESHOLD` (初期値: -100) 以下。
- **発生確率**: **100%（確定）**。通常エンカウントより優先。
- **勝利**: その後目的地へ到着。シェアテキスト `share_text` をレスポンスに含め、フロントエンドでシェアダイアログを表示する。
  - シェアテキスト例: `「名声が地に落ち賞金首として狙われたが、刺客を返り討ちにしてやった。 #Wirth_Dawn #悪名辟く」`
- **敗北ペナルティ**: 死亡（Vitality 0）は免除されるが、所持金の `BOUNTY_PENALTY_RATE` (初期値: 50%) を没収され、**ワールドマップ画面に出発地を選択した状態で強制送還**。

## 2. 首都への入場制限
王都レガリア、黄金都市イスハーク、神都「出雲」、天極城「龍京」への立ち入り制限。

### 2.1 通行許可証 (Pass)
- **アイテム定義**: `key_item` として定義。
- **価格**: `PASS_PRICE` (初期値: 20,000G)。首都以外の道具屋で販売。
- **有効期限**: `user_profiles.pass_expires_at` に「現在の `accumulated_days` + `PASS_DURATION_DAYS` (初期値: 365日)」を記録。
- **期限通知**: InnHeader にて残り30日未満の場合に警告バーを表示。

### 2.2 賄賂 (Bribe)
- **条件**: 名声が 0 未満の場合のみ選択可能。
- **費用**: `BRIBE_COST` (初期値: 10,000G)。
- **挙動**: 有効な許可証がない場合でも、賄賂を支払うことで強行突破（入城）が可能。

## 3. 名声（Reputation）システムの拡張

### 3.1 クエスト失敗ペナルティ
- **仕様**: バトル敗北やリタイアによるクエスト失敗時、その拠点での名声を `QUEST_FAIL_REP_PENALTY` (初期値: -5 〜 -10) 減少させる。

### 3.2 名声の自然減少 (Decay)
- **仕様**: 日次バッチにおいて、名声が `REP_DECAY_THRESHOLD` (初期値: 100) を超えている場合、毎日 `REP_DECAY_AMOUNT` (初期値: -5) ずつ自然減少。

### 3.3 クエスト受領条件の拡張
- **仕様**: `scenarios` テーブルに `max_reputation` (受注可能な名声の上限) を追加。
- **活用**: 「悪党限定クエスト」や「英雄限定クエスト」のマスタ設定。

## 4. 追加のゴールドシンク (Gold Sinks)

### 4.1 名声ロンダリング（身分洗浄）
- **場所**: **prosperity_level = 1（崩壊）の拠点の道具屋に「闇市」として出現**。崩壊拠点以外からは `POST /api/shop/launder` を呼び出せない（API側でprosperityチェックあり）。
- **費用**: `LAUNDERING_COST` (初期値: 100,000G)。
- **効果**: 全拠点の名声を 0 にリセット。
- **UI**: `prosperity_level = 1` の拠点でShopModalを開いた際、購入タブが「**闇市**」モードに切り替わり、以下の2枚カードが専用のダークデザイン（黒背景・赤枠）で表示される。
  - 「**禁術の秘薬**」（50,000G）: HP・MP全回復、最大HP恒久強化
  - 「**帳簿の改竄**」（100,000G）: 全拠点の名声を 0 にリセット（= 名声ロンダリング）
- **不正アクセス防止**: API側で `world_states.prosperity_level = 1` または `status = 'Ruined'` を確認。条件不一致の場合は `403 NOT_IN_RUINED_LOCATION` を返す。

### 4.2 英霊（残影）の摩耗
- **仕様**: **宿屋で休んでも、傭兵（NPC/残影）のHPは回復しない。**
- **回復手段**: バトル中の回復スキル等に限定。
- **目的**: 高額な再契約サイクルの創出。

### 4.3 祈り（Prayer）の強化
- **首都ボーナス**: 首都での祈りによる属性影響力を `CAPITAL_PRAYER_MULTIPLIER` (初期値: 2.0倍) に設定。
- **個人バフ (Blessing)**: 祈り実行時、次のバトル（クエスト）終了時まで以下のバフを付与。クエスト完了/失敗時に自動クリア。
  - `PRAYER_BUFF_HP_PCT` (初期値: 10%)
  - `PRAYER_BUFF_AP` (初期値: +1)
