# シェア機能（号外システム）および歴史アーカイブUI

## 1. 概要 (Overview)
プレイヤーの没入感を高め、かつX（旧Twitter）等のSNSへのオーガニックなシェアを促進するため、「歴史を紐解く」機能のUIを大幅に拡張し、特定のゲーム内体験をシェアできる導線（号外システム）を実装する。

## 2. シェア機能（号外システム）の要件とトリガー
ユーザーには「シェア」や「号外」というメタ的な言葉は使わず、「歴史の証人」「伝書鴉の報せ」といった世界観に沿った専用ダイアログを表示し、そこからX（旧Twitter）等への共有を行わせる。

### 2.1 世界情勢の変化（日次バッチ連動）
毎日の日次バッチ（`POST /api/cron/daily-update`）によって世界が変動した際、滞在中の拠点で何らかの情勢変化（繁栄度の増減、または支配国の変化）が起きたすべてのプレイヤーに対してシェアUIを展開する。

- **トリガー**: プレイヤーの `user_profiles.current_location_id` の拠点の `prosperity_level` または `alignment`（支配国）が、前回ログイン（確認）時から変化していた場合（ログイン時または宿屋トップ画面表示時に検知）。
- **シェアテキスト生成ロジック**: 変化した状態に応じてテキストを動的に生成する。
  - (繁栄度上昇) 「私が滞在する『{拠点名}』に活気が戻ってきた。この街の行く末を見届けよう。 #Wirth_Dawn #歴史の証人」
  - (繁栄度低下) 「私が滞在する『{拠点名}』に暗い影が落ち始めている…。 #Wirth_Dawn #歴史の証人」
  - (崩壊Lv1到達) 「ついに『{拠点名}』が崩壊の時を迎えた。黒煙が街を覆っている…。 #Wirth_Dawn」
  - (支配国変化) 「『{拠点名}』が{新支配国}の手に落ちた。私はその歴史の転換点に立ち会っている。 #Wirth_Dawn」

### 2.2 特別なクエストの達成（マスタ定義）
「特殊依頼（special）の最終話」などをプログラム側でハードコーディングするのではなく、マスターデータ（`scenarios` テーブル）側でシェアの有無を制御できるようにする。

- **DB改修**: `scenarios` テーブルに `share_text` カラム（TEXT, nullable）を追加。
- **トリガー**: クエスト完了API（`POST /api/quest/complete`）実行時、対象クエストに `share_text` が設定されている場合、リザルト画面に特大のシェアボタンを表示する。
- **シェアテキスト**: マスタに定義された `share_text` をそのまま使用（例: 「禁忌の森の最深部を踏破し、真実を知った…。 #Wirth_Dawn」）。
- **UGC初クリア (First Blood)**: `is_ugc: true` のクエストが世界で初めてクリアされた場合、システム側で自動的に以下のシェアテキストを生成し展開する。
  - 「名も無き旅人の依頼『{クエスト名}』を世界で初めて踏破した。 #Wirth_Dawn #未知への到達」

### 2.3 極限状態の体験（その他のシェア導線）
RPGならではのドラマチックな体験をシェアさせる。

- **英霊化（引退/死亡時）**:
  - **トリガー**: `POST /api/character/retire` 実行時、またはVitality枯渇による死亡時。
  - **テキスト**: 「我が名は{キャラクター名}。{年齢}歳の若さでこの世を去り、英霊として酒場に名を残す。誰か、私の残影を雇ってくれ。 #Wirth_Dawn #英雄の最期」
- **賞金稼ぎからの生還**:
  - **トリガー**: 名声-100以下のペナルティバトル（Bounty Hunters）に勝利した時。
  - **テキスト**: 「名声が地に落ち賞金首として狙われたが、刺客を返り討ちにしてやった。 #Wirth_Dawn #悪名轟く」

## 3. 「歴史を紐解く」UIの実装要件
宿屋トップ画面の「歴史（BookOpen）」アイコンから展開されるモーダル（またはフルスクリーン）UI。
古い古文書や手記をモチーフにしたリッチなデザインとし、以下の3つのタブ（またはセクション）で構成する。

### Tab 1: 私の歩み (Personal Chronicle)
プレイヤー個人の軌跡を日記風に振り返るエリア。
- **表示要素**:
  - 現在までの経過日数（`age_days`等から算出）とゲーム内カレンダー。
  - `user_completed_quests` の履歴リスト。単なるリストではなく「〇年〇の月 〇日：『{クエスト名}』を達成」と年代記風にフォーマットして表示。
  - **フレーバー統計**: 出禁になった拠点（名声0未満）の記録や、これまでの「裏切り」の回数などをテキストで添える。

### Tab 2: 世界の記憶 (World History)
自分がゲームを開始して以降の、世界情勢の変動履歴アーカイブ。
- **表示要素**:
  - 上記「2.1 世界情勢の変化」などで生成された号外（システムログ）の履歴一覧。
  - 「〇年〇月：華龍神朝が〇〇を併合」「〇年〇月：王都レガリアが崩壊」など、プレイヤーが関与・目撃した世界の歴史が時系列で並ぶ。

### Tab 3: 英霊の系譜 (Lineage of Heroes)
過去にプレイして引退・死亡した自分自身のキャラクターたちの記録（墓地データ）。
- **表示要素**:
  - 引退処理時に保存される「Graveyard Data（墓地スナップショット）」の読み込み。
  - 「初代：{名前}（享年{年齢}歳・死因: 寿命枯渇）」「二代：{名前}（享年{年齢}歳・死因: 自主引退）」といったリスト表示。
  - 最終滞在拠点や、こなしたクエスト数などの統計情報も併記する。

## 4. 追加タスク / データマイグレーション
1. **`scenarios` テーブル**: `share_text` (TEXT) カラムの追加。
2. **`world_states_history` と `user_world_views` テーブル (実装完了)**: 
   - 案Bを採用し、`world_states_history` テーブルを作成して日次バッチで変動ログを書き込み。
   - ユーザーの既読管理用として `user_world_views` を作成し、前回確認時以降の新しいニュースのみをシェアUIで展開する方式で実装済み。
3. **`graveyard` / `retired_characters` テーブル** (実装完了): 引退キャラクターのログ保存用テーブルを定義・実装済み。
