# シェア機能（号外システム）および歴史アーカイブUI

## 更新履歴
- **v15.1** (2026-04-04): シェア実装方式確定、未実装トリガー追加、歴史アーカイブUI詳細化
- **v15.0** (初版)

## 1. 概要 (Overview)
プレイヤーの没入感を高め、X（旧Twitter）等のSNSへのオーガニックなシェアを促進するため、「歴史を紐解く」機能のUIを大幅に拡張し、特定のゲーム内体験をシェアできる導線（号外システム）を実装する。

## 2. シェア実装の共通仕様

### 2.1 シェア方式
`https://twitter.com/intent/tweet?text={URLエンコードされたシェアテキスト}` のURL形成でXのブラウザを開く。

### 2.2 世界観UI
ユーザーには「シェア」や「号外」というメタ的な言葉は使わず、「伝書鴉に託す」という世界観に沿った専用ボタン名称を使用する。

---

## 3. シェア機能（号外システム）の要件とトリガー

### 3.1 世界情勢の変化（日次バッチ連動）
毎日の日次バッチ（`POST /api/cron/daily-update`）によって世界が変動した際、滞在中の拠点で情勢変化が起きたすべてのプレイヤーに対してシェアUIを展開する。

- **トリガー**: プレイヤーの `user_profiles.current_location_id` の拠点の `prosperity_level` または `alignment`（支配国）が、前回ログイン（確認）時から変化していた場合（ログイン時または宿屋トップ画面表示時に検知）。
- **実装**: `world_states_history` テーブルに変動ログを書き込み（実装済み）。`user_world_views` で既読管理（実装済み）。`ChronicleModal` コンポーネントで号外UIを表示（実装済み）。
- **シェアボタン**: ChronicleModal フッターに「伝書鴉に託す」ボタンを設置（実装済み）。
- **シェアテキスト生成ロジック** (`world-simulation.ts` で生成):
  - (繁栄度上昇) 「私が滞在する『{拠点名}』に活気が戻ってきた。この街の行く末を見届けよう。 #Wirth_Dawn #歴史の証人」
  - (繁栄度低下) 「私が滞在する『{拠点名}』に暗い影が落ち始めている…。 #Wirth_Dawn #歴史の証人」
  - (崩壊Lv1到達) 「ついに『{拠点名}』が崩壊の時を迎えた。黒煙が街を覆っている…。 #Wirth_Dawn」
  - (支配国変化) 「『{拠点名}』が{新支配国}の手に落ちた。私はその歴史の転換点に立ち会っている。 #Wirth_Dawn」

### 3.2 特別なクエストの達成（マスタ定義）

- **DB**: `scenarios` テーブルの `share_text` カラム（TEXT, nullable）で制御（実装済み）。
- **トリガー**: クエスト完了API（`POST /api/quest/complete`）実行時、対象クエストに `share_text` が設定されている場合、リザルト画面にシェアボタンを表示（実装済み）。
- **シェアテキスト**: マスタに定義された `share_text` をそのまま使用。
- **UGC初クリア (First Blood)**: `is_ugc: true` のクエストが世界で初めてクリアされた場合、システムが以下のテキストを自動生成（実装済み）:
  - 「名も無き旅人の依頼『{クエスト名}』を世界で初めて踏破した。 #Wirth_Dawn #未知への到達」

### 3.3 極限状態の体験

- **英霊化（引退/死亡時）**:
  - **トリガー**: `POST /api/character/retire` 実行時（`cause` パラメータで死因を区別）。
  - **テキスト**: 「我が名は{キャラクター名}。{享年（基本年齢 + accumulated_days/365）}歳の若さでこの世を去り、英霊として酒場に名を残す。誰か、私の残影を雇ってくれ。 #Wirth_Dawn #英雄の最期」
  - **Vitality枯渇死亡**: フロントエンドで `vitality <= 0` を検知した時点で、`cause='vitality_death'` として引退APIを呼び出し、同一シェアテキストでシェアUIを表示する。

- **賞金稼ぎからの生還**:
  - **トリガー**: 名声 ≤ -100 の状態での移動時エンカウント（Bounty Hunters）に**勝利**した時 (`encounter_type = 'bounty_hunter_ambush' & result = 'win'`)。
  - **テキスト**: 「名声が地に落ち賞金首として狙われたが、刺客を返り討ちにしてやった。 #Wirth_Dawn #悪名轟く」
  - **実装**: `POST /api/move/encounter-result` の win 処理で `share_text` をレスポンスに含める。フロントエンドで受け取りシェアダイアログを表示。

## 4. 「歴史を紐解く」UIの実装要件
宿屋トップ画面の「歴史（BookOpen）」アイコンから展開されるモーダル（HistoryArchiveModal）。
古い古文書や手記をモチーフにしたリッチなデザインとし、以下の3つのタブで構成する。

### Tab 1: 私の歩み (Personal Chronicle)
プレイヤー個人の軌跡を日記風に振り返るエリア。
- **表示要素**:
  - 現在までの経過日数（`accumulated_days`等から算出）とゲーム内カレンダー。
  - `user_completed_quests` の履歴リスト。単なるリストではなく「聖界暦 〇年〇の月 〇日：『{クエスト名}』を達成」と年代記風にフォーマットして表示。  
    ※ `completed_at`（リアル日付）を基点に求めるのではなく、クエスト達成時点の `accumulated_days` からゲーム内暦を算出すること（要APIからのデータ追加）。
  - **フレーバー統計**（Tab1下部に追加表示）:
    - 出禁になった拠点リスト: `reputations.score < 0` の `location_name` 一覧
    - クエスト途中放棄回数（※DBへの記録が必要。`user_profiles.abandon_count` カラムの追加は別途対応）

### Tab 2: 世界の記憶 (World History)
自分がゲームを開始して以降の、世界情勢の変動履歴アーカイブ。
- **表示要素**:
  - `world_states_history` テーブルの履歴一覧（実装済み）。
  - 「〇年〇月：{拠点名}が{新支配国}の手に落ちた」「〇年〇月：{拠点名}が崩壊」など時系列で表示。

### Tab 3: 英霊の系譜 (Lineage of Heroes)
過去にプレイして引退・死亡した自分自身のキャラクターたちの記録。
- **表示要素**:
  - `retired_characters` テーブルから取得（実装済み）。
  - 「初代：{名前}（享年{年齢}歳・死因: {cause_of_death}）」のリスト表示。
  - 最終滞在拠点・こなしたクエスト数等の統計情報を併記。

## 5. 追加タスク / データマイグレーション
1. **`scenarios` テーブル**: `share_text` (TEXT) カラム（実装完了）。
2. **`world_states_history` と `user_world_views` テーブル** (実装完了): 案Bを採用し実装済み。
3. **`graveyard` / `retired_characters` テーブル** (実装完了): 引退キャラクターのログ保存用テーブルを定義・実装済み。
4. **`user_completed_quests`**: クエスト達成時の `game_day`（ゲーム内経過日数）を記録するカラムの追加を検討（暦フォーマット表示に必要）。
5. **`user_profiles.abandon_count`**: クエスト途中放棄カウントカラム（後回し）。
