# 汎用クエスト（Normal Quests）設計・実装仕様書

## 1. 【Clean-Expert】テストデータ調査とクリーンアップ報告

過去のテスト用クエストおよびアイテムの残骸を特定しました。
プロジェクト内(`src/`)のTypeScript/TSXコードを解析した結果、クエストIDの直接的なハードコード（例: `6001` や `5999` のif分岐）は存在せず、マスタ駆動への移行に支障はありません。

### 削除対象のテストデータ
- **クエスト**: `5020:qst_sword_test`, `5998:qst_test_prerequisite`, `5999:qst_test_delivery`, `test_quest_001`
- **アイテム**: `3100:item_test_jewel`, `item_test_elixir`

### クリーンアップ実行SQLスクリプト
本番データの投入前に、以下のクエリを実行し環境を浄化してください。

```sql
-- 不要なテストクエストシナリオの削除
DELETE FROM scenarios WHERE slug IN ('qst_sword_test', 'qst_test_prerequisite', 'qst_test_delivery', 'test_quest_001');

-- 紐づくシナリオ進行状況（ユーザーデータ）の削除
DELETE FROM user_scenarios WHERE scenario_id IN (
    SELECT id FROM scenarios WHERE slug IN ('qst_sword_test', 'qst_test_prerequisite', 'qst_test_delivery', 'test_quest_001')
);

-- 不要なテスト用アイテムと所持データの削除
DELETE FROM items WHERE slug IN ('item_test_jewel', 'item_test_elixir');
DELETE FROM user_inventory WHERE item_id IN (
    SELECT id FROM items WHERE slug IN ('item_test_jewel', 'item_test_elixir')
);
```

## 2. 【Security-Expert】経済・名声バランス監査レポート

### ① 移動（Travel）エクスプロイトの防止
- **懸念**: 移動ノード(`travel`)のみのクエストで、報酬額が移動コストを大幅に上回るとリスクなしでの資金増殖（エクスプロイト）が発生する。
- **監査要件**: 
  - 単純な配達クエストの報酬上限は **[基本移動コスト + 50G]** とする。
  - 大金を稼げる輸送クエスト（報酬500G以上）には必ず **確定戦闘ノード（battle）** を組み込み、Vitality消費やカード消耗のリスクを負わせる。

### ② 名声（Reputation）条件のロック機能
- **監査要件**:
  - `min_reputation: "Famous"`（英雄的行動用）と `max_reputation: "Rogue"`（裏稼業用）を排他的に設定し、一方の遊び方をすると他方のクエストが消滅するトレードオフ機能を持たせる。

## 3. 【UIUX-Expert】テキストの2段階化とフレーバー演出

クエストボードの視認性とダークファンタジーの没入感向上のため、以下の2段階でテキストを書き分けます。
1. **short_description** (一覧用・約40文字): 「誰が」「何を」するかが一目でわかる端的な依頼書。
2. **full_description** (受領時詳細): 各国の腐敗や依頼者の泥臭い事情が垣間見える全テキスト。
