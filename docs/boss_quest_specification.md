# 特定フラグ出現型 強敵ボス討伐クエスト 設計・実装仕様書

## 1. 【Logic-Expert】20種のボス遭遇フラグと攻略ノード設計

メインシナリオから独立したエンドコンテンツとして、特殊な条件（フラグ）を満たしたプレイヤーのみが受諾できるボス討伐クエスト20種を定義します。

### 仕様要件（決定論的バトルの活用）
- **道中の消耗戦**: ボスに到達する前に必ず「道中の雑魚戦（`battle`ノード）」を3回挟みます。これにより、プレイヤーのHPとAPを適度に削り、デッキの回転（カード消耗）を強いるリソース管理戦とします。
- **ボスの行動パターン応用**: `turn_mod`（特定ターンでの大技発動）と、`hp_under`（HP低下時の発狂モード移行）アクションを組み込み、DPSチェックや防御タイミングの戦略性を生み出します。

## 2. 【Security-Expert】バランス監査とエクスプロイト防止

### ① 30ターン敗北と「無限回復・永続遅延」の検証
- **ボスの回復制限**: 堕天使や大司教などの回復スキルには、`cooldown: 5`（5ターンに1回しか使えない）または `use_limit: 3`（戦闘中3回まで）の制限を課します。（※現行仕様でcooldownが未実装の場合は、`turn_mod: 5`を用いて「5ターンごとの行動枠」でのみ回復を許可する設定とします）。これにより、ボスの無限回復による30ターンのタイムオーバー敗北（プレイヤー側のリソース切れ負け）を防ぎます。
- **HPとDEFの適正化**: 3連戦でカードを消耗した状態で挑むことを考慮し、ボスのHPは「理論上最高火力の10〜15ターン分」、DEFは「プレイヤーの中央値攻撃力を貫通できるが、防御系バフがあれば0に抑えられる値」に調整します。過度なDEFによる「常時ダメージ0」状態を禁止します。

### ② 報酬の多重獲得防止（1回限り制限）
- **前提条件の活用**: 一度討伐したボスクエストが再度掲示板に並ばないよう、シナリオマスタの `requirements`（`completed_quest`の非保持など）、あるいは `is_repeatable: false` のフラグをシステム側で保証します（現在クエストシステムは `user_scenarios` によってクリア状態が管理されています）。
- 特大報酬（8000Gなど）を含むドラゴン討伐等は、絶対に周回不可の設計とします。

## 3. 【UIUX-Expert】テキストの2段階化とSNSシェア機能

### テキストの書き分けと「share_text」の導入
エンドコンテンツとしての緊張感を高めるため、討伐依頼テキストを重厚なダークファンタジー調で描きます。また、最難関ボス（ドラゴン、麒麟、悪魔、天使など）には、討伐完了時に表示できるSNSシェア用テキスト（`share_text`）を定義します。

**【テキストサンプル 1: デザートドラゴン (ID: 8023)】**
- **Short**: `[討伐特務] 砂漠の覇者、デザートドラゴンが覚醒した。死を恐れぬ者のみ挑め。`
- **Full**: `「空が赤く燃えている……熱砂の深淵より『古き竜』が眼を覚ましたのだ。あのような巨大な絶望を前に、軍隊など何の意味もない。傭兵よ、お前が持つその『逆鱗』が竜を惹きつけている。我々の街が灰になる前に、あの空の火を消してこい！」`
- **Share**: `砂海の絶対君主、デザートドラゴンを討ち取った。この熱砂に私の名が深く刻まれるだろう。「次は、どの神を殺そうか？」 #Code_Wirth_Dawn #竜殺し`

**【テキストサンプル 2: 降臨せし天使 (ID: 8022)】**
- **Short**: `[極秘討伐] 狂信をばら撒く厄災の天使を撃墜せよ。`
- **Full**: `「天より降り立ったそれは、我々を救う使者などではなかった。美しき光で人々の理性を奪い、生きたまま天界の苗床に変えようとしているのだ。頼む、あの冒涜的な翼をもぎ取り、穢れた地上へ墜としてくれ。」`
- **Share**: `天より降りし破壊の御使いを、この刃で地に墜としてやった。我々には神の救済など必要ない。 #Code_Wirth_Dawn #神殺しの傭兵`

**【テキストサンプル 3: 腐敗枢機卿 (ID: 8012)】**
- **Short**: `[暗殺] 表の顔は聖職者、裏は人身売買の元締め。枢機卿の首を獲れ。`
- **Full**: `「教会の奥深くで、あの太った豚は神の名を騙りながら人々を売り飛ばしている。光の届かぬ場所で手を汚してきたお前たちにしか頼めない仕事だ。あの男の寝首を掻き、地獄へ案内してやってくれ。」`

これらの設計に沿って、ボスデータ(enemies.csv)、ノード展開(scenarios/*.csv)、シナリオ出現フラグ(quests_special.csv)を作成・実装していきます。
