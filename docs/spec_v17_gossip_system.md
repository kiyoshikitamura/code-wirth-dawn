# 仕様書 v17: 街の噂話 (Gossip) システム仕様

## 1. 概要と目的
「街の噂話」は、各拠点においてプレイヤーが収集できる情報の集合体です。拠点画面の施設グリッド（「ステータス」ボタンの隣）に配置される「街の噂話」ボタンからアクセス可能です。
プレイヤーに世界観への没入やヒントを提供し、テキストを通じてリアルタイムに変化する世界を体感してもらうことを目的とします。

## 2. デザイン・UI
- JRPG風のセピア調モーダルを採用し、ロールプレイ感を強調します。
- **演出**: タイプライター風の文字表示アニメーション（`▍`のカーソル点滅付き）を採用し、「情報が書き込まれる」「人から話を聞く」ような表現をしています。複数の情報が同時に表示される場合、テキスト長から遅延を計算し、**上から順番に一つずつタイピングが流れる**ようにシーケンシャルな表示制御を行っています。
- 各タブの情報はモーダル下部の「もっと情報を聞く（更新）」ボタンで新たに情報を取得することができます。

## 3. タブ構成とデータ連携

「街の噂話」は4つのタブで構成されており、それぞれ異なるデータソース情報を表示します。
全タブ共通の仕様として、**各種情報は最大3件までランダムに抽出して表示**されます。これにより更新ボタンを押すたびに異なる情報を得られる可能性があります。

### 3.1. 噂話 (News)
- **内容**: プレイヤーが所属する世界の情勢変化や大きな出来事（拠点支配権の変動、大規模な討伐記録など）。
- **データソース**: `world_states_history` テーブル
- **表示件数**: 直近履歴から最大3件をランダム表示。
- **機能**: 新着の未読情報がある場合は、未読バッチが付与されます。（未読カウントは `user_world_views` の `last_seen_history_id` との比較で判定）。

### 3.2. こぼれ話 (Lore)
- **内容**: 街の住人が話す、世界観に関するフレーバーテキスト、および**闇市の噂**。
- **データソース**: 
  - `rumors` テーブル (`category = 'lore'`、または全拠点共通 `nation_tag = null` のもの)
  - `items` テーブル (`is_black_market = true` のアイテム: 闇市の品として自動で噂話化されて混ざる)
- **詳細**: `{loc_name}` や `{nation_name}` のプレースホルダーが使用されており、現在プレイヤーがいる拠点名や統治国名がサーバー側で動的に差し替えられます。
- **表示件数**: 該当する全件から最大3件をランダム表示。

### 3.3. 路地裏 (Secret)
- **内容**: 隠された情報、特定の条件を満たすレアクエスト（サブクエスト類）の情報。
- **データソース**: 
  - `scenarios` テーブル (`quest_type = 'special'` かつ `is_urgent = true` かつ `slug` が `main_` 以外)
- **詳細**:
  - メインシナリオのクエストが流れてしまうのを防ぐため、内部IDが `main_` となっているものは除外されます。
  - `scenarios.requirements`（参加条件）を意図的に曖昧な表現（例：「相応の腕前がある者」「ローランドの縁者」など）に変換し、ヒントとして添えつつ、**クエストタイトルを明記**（「『〇〇』という依頼が出回っているらしい……」）して提示します。
- **表示件数**: 出現中のレアクエストから最大3件をランダム表示。

### 3.4. 酒場 (Tavern)
- **内容**: 現在同じ拠点に滞在している他のプレイヤー冒険者や霊体（Shadow）の情報。
- **データソース**:
  - `party_members` テーブル (`origin_type` が `shadow` 系 かつ 現在持ち主がいないもの)
  - `user_profiles` の `current_location_id`（滞在中の他プレイヤー数を算出、共鳴人数として表示）
- **表示件数**: Shadow枠を含め最大3件をランダムで提示。

## 4. 更新ロジック
「もっと情報を聞く」をクリックすることで `GET /api/gossip` へリクエストが送られ、対応するタブの情報が再取得および再シャッフルされます。各種3件ずつのランダム表示により、何度か聞くことで様々な情報が閲覧できるように設計されています。
