# 仕様書 v17: 街の噂話 (Gossip) 掲示板システム & 自己紹介拡張仕様

## 1. 概要と目的
「街の噂話」は、各拠点画面からアクセス可能なモーダルで動作するプレイヤー参加型の掲示板（BBS）システムです。
プレイヤーがリアルタイムに情報を発信し合うだけでなく、世界の情勢変化や他のプレイヤーが達成した偉業がシステムアカウント「世界の観測者」によって自動的に布告され、常に流動的に変化する世界観を提示することを目的とします。
これに伴い、従来の4タブ形式による噂話表示仕様およびそれに付随するデータソース、スクリプト群はクリーンアップ（完全廃止）されます。

---

## 2. デザイン・UI
- **掲示板形式**: X (旧Twitter) のようなタイムライン型モーダルレイアウト。
- **描画演出**: タイプライター風の文字表示アニメーション（カーソル点滅等）は全面的に不要とし、即時描画（またはフェードイン）を行います。
- **最上段固定（ピン留め）エリア**:
  - 最新のシステムメッセージ1件を最上段に固定。
  - 神秘的なダークトーン（青紫色や金色のグラデーション）の背景で通常投稿と明確に差別化します。
  - 投稿者名は「世界の観測者」、アバター画像は宝石をモチーフにした専用グラフィック（`/images/icons/observer_gem.png`）を使用します。
- **タイムラインエリア**:
  - 投稿者名、称号（通り名）、アバター、本文、投稿日時（リアル時間）、書き込んだ時点の滞在拠点を表示。
  - 「滞在拠点」と「投稿日時」は投稿カードの左下に綺麗にレイアウトします。
  - アバターまたは名前をタップすると、該当プレイヤーの「自己紹介ポップアップ」を起動します。
- **スクロールと無限読み込み**:
  - 初期表示で30件をロードし、フリック・スクロールで一番下までスクロールすると次の30件を追加ロード（無限スクロール形式）。
  - PC環境向けには、上品なセピア調またはダークゴールド調のカスタムスクロールバー（Vanilla CSS）を適用します。
- **フローティングボタン（FAB）**:
  - モーダル画面 of 右下に固定された「＋」ボタンを配置。
  - タップすると「噂を書き込む」ポップアップが立ち上がり、テキストエリア（140文字上限、残文字カウント）、投稿（送信）ボタン、キャンセル（戻る）ボタンを提供。

## 3. データベース & API設計

### 3.1 `gossip_posts` テーブル
- `id` (bigint, PK, identity): 投稿の一意なID。
- `user_id` (uuid, FK user_profiles.id, nullable): 投稿ユーザーID。システムメッセージの場合は NULL。
- `name` (text, not null): 投稿者の名前（投稿時のスナップショット）。
- `epithet` (text, nullable): 投稿者の称号（投稿時のスナップショット）。
- `avatar_url` (text, nullable): 投稿者のアバターアイコンURL。
- `content` (varchar(140), not null): 噂本文（最大140文字）。
- `location_id` (uuid, FK locations.id, nullable): 投稿時の滞在拠点ID。
- `location_name` (text, nullable): 投稿時の滞在拠点名（スナップショット）。
- `is_system` (boolean, default false): システムによる自動投稿フラグ。
- `created_at` (timestamptz, default now()): 投稿されたリアル日時。

#### インデックス
- `gossip_posts_created_at_idx` ON `gossip_posts(created_at DESC)` (追加読み込み・ソート高速化用)

#### RLSポリシー (Row Level Security)
- `SELECT`: 認証済みユーザーであれば全件閲覧可能。
- `INSERT`: 認証済みユーザーは `user_id = auth.uid()` かつ `is_system = false` の条件で挿入可能。`content` が140文字以下であることを検証。

### 3.2 API エンドポイント
- `GET /api/gossip`
  - パラメータ: `limit` (デフォルト30), `offset` または `cursor` (追加読込用)。
  - 返却データ:
    - `pinned_system_post`: 最も新しい `is_system = true` のレコード1件。
    - `posts`: 通常のタイムライン投稿リスト（ピン留めメッセージを除外したもの）。
- `POST /api/gossip`
  - 送信データ: `{ content: string }`
  - バリデーション:
    - 140文字以下であること。
    - 直前投稿から**「30秒」**経過していること（30秒以内の連続投稿は `429 Too Many Requests` エラー）。
    - 禁止ワードフィルタ（誹謗中傷、ハラスメントワード等）に抵触していないこと（抵触時は `400 Bad Request` エラー）。
  - 送信者のプロフィール (`user_profiles`) から最新の `name`、`title_name`、`avatar_url`、`current_location_id` と拠点名を解決して `gossip_posts` に挿入。

## 5. 自己紹介ポップアップ (`SimpleUserProfilePopup`) の要素拡張
- ポップアップ表示要素を以下の6点に拡張します。
  - **称号 (epithet / title_name)**
  - **ユーザー名 (name)**
  - **ユーザーアイコン (avatarUrl)**
  - **レベル (level)**
  - **ゲーム内年齢 (age)**
  - **自己紹介文 (introduction)**
- **データ取得方法**:
  - ランキングやタイムライン上のカードをクリックした場合は、`/api/profile?profileId=[ID]` から非同期で現在アクティブなプロフィールを取得し、ポップアップへ渡します。
  - 酒場（`TavernModal.tsx`）の傭兵NPCをクリックした場合は、`ShadowSummary` に格納されている `level` および mapping したデータをポップアップへ渡します。

## 6. 旧仕様のクリーンアップ (Cleanup)
掲示板（BBS）移行に伴い、不要となる旧仕様のコード資産およびデータを完全に削除・整理します。
1. **DBテーブル廃止**:
   - `rumors` テーブルの削除。
2. **スクリプト廃止**:
   - `scripts/sync_rumors.ts` の削除。
3. **API側の旧機能削除**:
   - `src/app/api/gossip/route.ts` 内の旧4タブ（News, Lore, Secret, Tavern）向けのデータ抽出クエリ、および `formatRequirementsHint` ヘルパーを全削除。
   - `src/app/api/init-page/route.ts` 内の `gossip_data` pre-fetch における旧4タブ用のクエリ・組み立てコードを全削除し、新しいBBSタイムラインのデータ取得へ置き換え。
4. **UI側の旧演出・ボタン削除**:
   - `src/components/world/GossipModal.tsx` 内の旧タイプライターコンポーネント（`TypewriterCard`, `TypewriterCardWithNext`, `SequentialCards`）、タブ切り替え定数 `TABS`、酒場誘導ボタン（`onOpenTavern` 等）を完全削除。
