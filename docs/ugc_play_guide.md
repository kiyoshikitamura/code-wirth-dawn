# 冒険者の掲示板 — UGCクエストガイド

ようこそ、冒険者の掲示板へ。
この掲示板は、世界中の冒険者たちが自ら創り上げたオリジナルクエストを共有し、挑戦できる場所です。
誰かの物語を体験するもよし、あなた自身が物語を紡ぐもよし。
本ガイドでは、掲示板の使い方と、クエスト制作の手引きを解説します。

---

## 1. UGCクエストとは

### 1.1 概要
* UGCクエスト（冒険者投稿クエスト）とは、プレイヤー自身がテンプレートを使って制作し、他の冒険者に向けて公開するオリジナルクエストです。
* 独自のストーリー、カスタムエネミー、オリジナルアイテム、オリジナルスキルカード、同行NPCを自由に組み合わせて、あなただけの冒険を創ることができます。
* UGCクエストは通常の公式クエストとは**独立したシステム**で運営されています。報酬や進行ルールが一部異なりますので、本ガイドをご確認ください。

### 1.2 どこで遊べるか
* UGCクエストは「**名もなき旅人の拠所**」のギルドにある「**クエストボード**」からのみ受注できます。
* 通常の拠点のギルド（クエストボード）にはUGCクエストは掲載されません。UGCクエストを遊びたい場合は、名もなき旅人の拠所へ向かいましょう。

### 1.3 どこで作れるか
* クエストの制作は「**名もなき旅人の拠所**」に滞在しているとき、「**クリエイターズ工房**」メニューから行えます。
* テンプレートファイルのダウンロード、編集済みテンプレートのインポート、作品の管理と公開申請をすべてこの画面で行います。

## 2. UGCクエストを遊ぶ

### 2.1 クエストボード
名もなき旅人の拠所のギルド（クエストボード）を開くと、他の冒険者が公開しているUGCクエストの一覧が表示されます。通常の拠点ではギルドに公式クエストが表示されますが、名もなき旅人の拠所のギルドはUGCクエスト専用です。

#### 検索
* 画面上部の検索バーに文字を入力すると、**クエスト名**または**制作者名**で絞り込みができます。
* 並び替えは「新着順」「人気順」から選べます。

#### クエストカードの見方
各クエストは以下の情報が表示されます。

| 表示項目 | 説明 |
|---------|------|
| クエスト名 | 制作者が付けたタイトル |
| 制作者名 | クエストを公開した冒険者の名前 |
| 推奨レベル | 挑戦に適したキャラクターレベルの目安 |
| 種別 | 討伐 / 納品 / 政治 / 迷宮 / その他 |
| 難易度 | 1〜10の難易度評価 |
| プレイ回数 | これまでに挑戦された回数 |
| クリア回数 | これまでにクリアされた回数 |

### 2.2 受注条件
* **レベル制限**: UGCクエストを受注するには、キャラクターが**レベル5以上**である必要があります。レベル5未満でもクエスト一覧の閲覧と詳細の確認は可能ですが、受注ボタンは無効化されます。
* **アライメント条件**: クエストごとに、制作者が設定したアライメント条件がある場合があります。例えば「秩序率30%以上」など、特定のアライメント傾向を持つ冒険者のみが受注できるクエストがあります。条件はクエスト詳細画面で確認できます。
* **同時受注制限**: 通常クエストと同様に、同時に進行できるクエストは1つまでです。公式クエストまたはUGCクエストのいずれかを進行中は、他のクエストを受注できません。

### 2.3 クエスト進行
* UGCクエストの進行方法は通常のクエストと同じです。テキストを読み、選択肢を選び、戦闘を行いながらシナリオを進めます。
* 制作者がカスタムした独自のエネミーとの戦闘、オリジナルNPCの同行、独自のBGMや効果音が楽しめます。
* **クエスト中の制限**も通常と同様です。デッキの変更や装備の着脱はできません。

### 2.4 報酬について
UGCクエストの報酬は、ゲーム経済のバランスを守るため、通常クエストとは異なるルールが適用されます。

| 報酬種別 | 内容 |
|---------|------|
| **ゴールド** | クリア時に一律 **50G** が支給されます（制作者による変更不可） |
| **経験値** | クリア時に一律 **30 EXP** が支給されます（制作者による変更不可） |
| **名声** | 変動しません |
| **アライメント** | 変動しません |
| **カスタムアイテム** | 制作者が用意した独自のアイテム（最大3個）が入手できます |
| **カスタムスキル** | 制作者が用意した独自のスキルカード（最大1枚）が入手できます |

* カスタムアイテムは通常アイテムと同様にインベントリに保存されますが、ショップでの売却価格は一律 **1G** です。
* カスタムスキルカードは通常のスキルと同様にデッキに組み込んで使用できます。
* 加齢、VIT摩耗、パーティメンバーの損耗は通常クエストと同一のルールが適用されます。

### 2.5 放棄
* UGCクエストは途中で放棄できます。通常クエストと同じく、放棄時にはペナルティ（VIT-1、名声低下）が発生します。

## 3. UGCクエストを作る — 制作の流れ

### 3.1 全体フロー

```
① テンプレートをダウンロード
   ↓
② お好みのテキストエディタで編集
   ↓
③ バランス計算ツールでバランスを確認（任意）
   ↓
④ 画像やBGMを用意してアップロード（任意）
   ↓
⑤ 編集したテンプレートをインポート
   ↓
⑥ バリデーション結果を確認・修正
   ↓
⑦ 下書き保存
   ↓
⑧ テストプレイでクリアを確認
   ↓
⑨ 公開申請
   ↓
⑩ 運営の審査を経て公開
```

### 3.2 ① テンプレートのダウンロード
* クリエイターズ工房の「テンプレート」タブで、テンプレートファイルをダウンロードします。
* テンプレートの種類は以下の通りです。

| テンプレート | JSON | MD | 用途 |
|------------|:----:|:--:|------|
| クエストテンプレート | ✅ | ✅ | シナリオフロー全体を制作する |
| エネミーテンプレート | ✅ | ✅ | 独自の敵キャラクターを個別に作成 |
| アイテムテンプレート | ✅ | ✅ | 独自の消耗品や素材を作成 |
| スキルカードテンプレート | ✅ | ✅ | 独自の戦闘スキルを作成 |
| NPCテンプレート | ✅ | ✅ | 独自の同行キャラクターを作成 |

* すべてのテンプレートが **JSON形式** と **MD（Markdown）形式** の両方でダウンロードできます。
* 初めてクエストを作る方は、**MDテンプレート**から始めることをお勧めします。コメント付きのガイドが含まれており、日本語で直感的に記述できます。
* **スマートフォンユーザー**は、「🤖 AIで作成」ボタンから対話型AI向けプロンプトをコピーして、AI（ChatGPT等）にテンプレートを作ってもらうこともできます（詳しくは第3.4章を参照）。

### 3.3 ② テンプレートの編集
* ダウンロードしたファイルをお好みのテキストエディタ（メモ帳、VSCode、サクラエディタなど）で開いて編集します。
* MDテンプレートの詳しい書き方は「第4章 MDテンプレートの書き方」をご覧ください。

### 3.4 🤖 AIプロンプトガイドを使う（モバイル推奨）

テキストエディタでの編集が難しい場合（特にスマートフォンユーザー）は、**AIプロンプトガイド**を活用できます。

```
① テンプレートタブで「🤖 AIで作成」ボタンをタップ
   ↓
② モーダルが開き、AI向けプロンプトが表示される
   ↓
③ 「📋 クリップボードにコピー」をタップ
   ↓
④ ChatGPT / Gemini / Claude 等にプロンプトを貼り付け
   ↓
⑤ AIとの対話でテンプレートを完成させる
   ↓
⑥ 完成したJSON/MDをコピー
   ↓
⑦ 「インポート」タブからテキストを貼り付けてインポート
```

* プロンプトには、フィールド定義・制約値・バランスルール・出力形式の指定がすべて含まれているため、AIが正確なテンプレートを生成できます。
* JSON形式とMD形式のどちらでAIに出力させるかを選択できます。
* AIが生成したテンプレートも、インポート時に通常のバリデーションが実行されるため、安全です。

> 💡 **ヒント**: 簡易ビルダー（「簡易作成」タブ）は、プログラミング経験がなくてもステップごとにクエストを組み立てられるUI付きの作成モードです。まずは簡易ビルダーを試してみて、より自由度が欲しくなったらテンプレートやAIプロンプトガイドに移行するのがおすすめです。

### 3.5 ③ アセット（画像・音声）のアップロード
* クエストで使用する独自の画像やBGM・効果音がある場合は、クリエイターズ工房からアップロードしておきます（詳細は第6章）。
* **画像やBGMは任意です。** 用意しなくても公式素材（背景・BGM）を使ってクエストを制作できます。

### 3.6 ④ テンプレートのインポート
* クリエイターズ工房の「インポート」タブから、2つの方法でテンプレートを取り込めます。

| インポート方法 | 推奨ユーザー | 操作 |
|--------------|------------|------|
| 📋 **テキスト貼り付け**（デフォルト） | モバイル・AIで作成した方 | AIが生成したテンプレートをコピー → テキストエリアに貼り付け |
| 📁 **ファイル選択** | PCでエディタを使って編集した方 | ダウンロードしたファイルを選択 |

* **テキスト貼り付け**モードでは、JSON/MD形式がテキスト内容から自動判定されます（`---` で始まるとMD、`{` で始まるとJSON）。
* ファイル名を意識する必要がないため、AIの出力をそのままコピー＆ペーストできます。
* インポート時にリアルタイムでバリデーション（内容チェック）が実行され、問題があればエラーが表示されます。

### 3.7 ⑤ バリデーション結果の確認
* バリデーション結果は3種類です。

| 種別 | 意味 |
|------|------|
| ✅ 成功 | 問題なし |
| ⚠️ 警告 | クエストは作成可能だが、改善の余地あり（画像未指定など） |
| ❌ エラー | 修正しないとインポートできない致命的な問題 |

* エラーには行番号が表示されるので、テンプレートの該当箇所を修正してから再度インポートしてください。

### 3.8 ⑥ 下書き保存
* バリデーションを通過したら「下書き保存」ボタンで保存します。
* 下書きは何度でも上書きできます。テンプレートを修正して再インポートすると、最新の内容で更新されます。

### 3.9 ⑦ テストプレイ
* 公開申請の前に、**自分自身でクエストをプレイしてクリア**する必要があります。
* テストプレイでは、VIT消費や敗北ペナルティは発生しません。報酬も付与されません。
* テストプレイをクリアすると「テスト済み」マークが付き、公開申請が可能になります。
* **テンプレートを再インポートして内容を変更すると、テスト済みマークはリセットされます。** 再度テストプレイが必要です。

### 3.10 ⑧ 公開申請
* テスト済みの作品は「公開申請」ボタンから審査に提出できます。

### 3.11 ⑨ 審査と公開
* 提出された作品は運営チームによる審査を経て公開されます。
* 審査では、不適切なコンテンツや著作権に問題がないかを確認します（詳細は第10章）。
* 承認されるとギルドのクエストボードに掲載され、他の冒険者が挑戦できるようになります。
* 却下された場合は理由が通知されます。内容を修正して再申請することができます。

## 4. MDテンプレートの書き方

MDテンプレートは、テキストファイルの上部に「ヘッダー情報」、下部に「シナリオ本文」を記述します。

### 4.1 ヘッダー情報

ファイルの先頭に `---` で囲んだブロックにクエストの基本情報を記述します。

```yaml
---
version: "1.0"
type: quest

title: "古き森の守護者"
short_description: "森の奥に棲む獣を退治してほしい"
full_description: |
  この森には古くから守護獣が棲んでいた。
  冒険者よ、どうかあの獣を鎮めてくれ。
client_name: "森の長老"
scenario_type: "Subjugation"
difficulty: 3
rec_level: 10
days_success: 1
days_failure: 1

# 受注条件（省略可。設定したいものだけ記述してください）
conditions:
  min_align_order_pct: 30

# 報酬（省略可。詳細は第7章を参照してください）
rewards:
  items:
    - name: "守護獣の牙"
      type: "trade_good"
      description: "鎮められた守護獣から得た牙。"
---
```

| フィールド | 説明 | 必須 |
|-----------|------|------|
| `version` | `"1.0"` と記述してください | ✅ |
| `type` | `"quest"` と記述してください | ✅ |
| `title` | クエスト名（30文字以内） | ✅ |
| `short_description` | 一覧に表示されるキャッチコピー（40文字以内） | ✅ |
| `full_description` | 詳細画面に表示されるフレーバーテキスト | — |
| `client_name` | 依頼人の名前（省略すると「謎の依頼人」） | — |
| `scenario_type` | `Subjugation`（討伐）/ `Delivery`（納品）/ `Politics`（政治）/ `Dungeon`（迷宮）/ `Other`（その他） | — |
| `difficulty` | 難易度（1〜10） | — |
| `rec_level` | 推奨レベル（1〜50） | — |
| `days_success` | クリア時の経過日数（1〜10） | — |
| `days_failure` | 失敗時の経過日数（1〜10） | — |

### 4.2 シナリオ本文 — ノードの書き方

ヘッダーの `---` の後に、シナリオをノード単位で記述します。
各ノードは `## ノード 番号: タイトル（タイプ）` の見出しで始めます。

#### テキストノード（会話・ナレーション）

最も基本的なノードです。プレイヤーにテキストを表示します。

```markdown
## ノード 1: 依頼の始まり（テキスト）
**話者**: 森の長老
**画像**: ugc://images/scenarios/elder.webp
**背景**: bg_forest_day
**BGM**: bgm_quest_calm

> 旅の方よ、よく来てくれた。
> この森の守護獣が暴走しておる。
```

* `**話者**`: 発言者の名前（テキストの横に表示されます）
* `**画像**`: 話者の立ち絵画像（任意）
* `**背景**`: 背景画像のキー（公式背景またはカスタム画像）
* `**BGM**`: BGMのキー（公式BGMまたはカスタム音楽）
* `**SE**`: 効果音のキー（任意）

テキスト本文は `>` で始めることを推奨しますが、通常のテキストでも構いません。

#### 選択肢

プレイヤーに選択肢を提示して分岐させるには、`**選択肢**:` の後にリスト形式で記述します。

```markdown
## ノード 2: 分かれ道（テキスト）

道が二手に分かれている。どちらへ進む？

**選択肢**:
- [左の道を行く] → ノード 3
- [右の道を行く] → ノード 4
```

#### 自動進行

選択肢がないノードは、`→ ノード N` で次のノードを指定します。省略した場合は次の番号のノードに自動で進みます。

```markdown
## ノード 3: 左の道（テキスト）

穏やかな小道を進む。

→ ノード 5
```

#### バトルノード

カスタムエネミーとの戦闘を発生させます。

```markdown
## ノード 5: 守護獣戦（バトル）
**背景**: ugc://images/scenarios/forest_battle.webp
**BGM**: ugc://audio/bgm/boss_theme.mp3

**エネミー**:
  名前: 呪われた守護獣
  レベル: 10
  HP: 200
  ATK: 12
  DEF: 8
  スキル: [heavy_blow, heal_self]
  画像: ugc://images/enemies/guardian.webp
  フレーバー: 呪いにより凶暴化した守護獣。
```

勝利すると次のノードに進みます。敗北するとクエスト失敗です。

#### NPC加入ノード

戦闘中に味方として同行するNPCをパーティに加えます。

```markdown
## ノード 3: マリアの合流（NPC加入）

**NPC**:
  名前: 森の狩人マリア
  レベル: 12
  ATK: 8
  DEF: 5
  耐久度: 100
  カバー率: 15
  AI: striker
  スキル: [arrow, double_slash]
  画像: ugc://images/npcs/maria.webp
  護衛対象: false
```

* `護衛対象: true` にすると、そのNPCの耐久度が0になった時点でクエスト失敗になります。

#### NPC離脱ノード

```markdown
## ノード 8: マリアとの別れ（NPC離脱）

「ここから先は一人で大丈夫だな。武運を祈る。」
```

#### 納品ノード

プレイヤーの所持品から指定アイテムを消費します。持っていなければクエスト失敗です。

```markdown
## ノード 4: 素材の納品（納品）
**納品アイテム**: item_potion_s × 3
```

#### ランダム分岐ノード

確率で異なる結果に分岐します。

```markdown
## ノード 6: 罠の回避（ランダム分岐）
**確率**: 60

うまく罠を避けられるか…？

**選択肢**:
- [成功] → ノード 7
- [失敗] → ノード 8
```

`**確率**: 60` は60%の確率で1番目（成功）に分岐します。

#### 罠ノード

プレイヤーにダメージを与えます。

```markdown
## ノード 7: 毒の沼（罠）
**ダメージ**: 20%

足元から毒沼が噴き出した！
```

`**ダメージ**: 20%` はプレイヤーの最大HPの20%のダメージを与えます。

#### 成功ノード・失敗ノード

クエストの終了を表します。**すべてのクエストには少なくとも1つの成功ノードが必要です。**

```markdown
## ノード 10: めでたしめでたし（成功）

長老が深く頭を下げた。
「ありがとう、旅の人よ。」
```

```markdown
## ノード 11: 無念の撤退（失敗）

力及ばず、撤退するしかなかった…。
```

### 4.3 使用可能な公式背景キー

テンプレート内で `**背景**:` に指定できる公式背景の一覧です。カスタム画像がない場合はこちらをご利用ください。

| キー | 説明 |
|------|------|
| `bg_road_day` | 街道（昼） |
| `bg_forest_day` | 森（昼） |
| `bg_forest_night` | 森（夜） |
| `bg_deep_forest` | 深い森 |
| `bg_cave` | 洞窟 |
| `bg_ruins` | 遺跡 |
| `bg_castle` | 城 |
| `bg_town` | 街並み |
| `bg_wasteland` | 荒野 |
| `bg_mountain` | 山岳 |

### 4.4 使用可能な公式BGMキー

| キー | 説明 |
|------|------|
| `bgm_quest_calm` | 穏やかな冒険曲 |
| `bgm_quest_tense` | 緊迫した冒険曲 |
| `bgm_battle` | 通常戦闘曲 |
| `bgm_battle_boss` | ボス戦闘曲 |
| `bgm_quest_clear` | クエストクリア曲 |

### 4.5 使用可能なエネミースキル

バトルノードのエネミーに設定できるスキルの代表例です。テンプレートのダウンロード画面で完全なリストを確認できます。

| スキル名 | 説明 |
|---------|------|
| `tackle` | 体当たり（×1倍ダメージ） |
| `heavy_blow` | 強打（×2倍ダメージ） |

## 5. カスタムアセットの作り方

### 5.1 エネミーテンプレート
クエスト内で使用する独自のエネミーを個別に作成できます。JSON形式で以下のように記述します。

```json
{
  "$template": "wirth-dawn-ugc",
  "version": "1.0",
  "type": "enemy",
  "enemy": {
    "name": "呪われた守護獣",
    "level": 10,
    "hp": 200,
    "atk": 12,
    "def": 8,
    "skills": ["heavy_blow", "heal_self"],
    "image_url": "ugc://images/enemies/guardian.webp",
    "flavor_text": "呪いにより凶暴化した守護獣。"
  }
}
```

**TP（脅威度ポイント）制限**: エネミーのステータスは「TP制」で制限されます。エネミーのレベルに応じたTP上限内にステータスを収める必要があります。

| TP上限の計算 | 10 + レベル × 5 |
|-------------|----------------|
| Lv5のエネミー | TP上限 35 |
| Lv10のエネミー | TP上限 60 |
| Lv30のエネミー | TP上限 160 |

| ステータス | TPコスト |
|-----------|--------|
| HP +1 | 1 TP |
| ATK +1 | 2 TP |
| DEF +1 | 2 TP |
| 全体攻撃スキル（AoE） | 20 TP |
| VIT吸収スキル | 30 TP |
| その他スキル | 0 TP |

例: Lv10のエネミー（TP上限60）で HP200(=200) + ATK12(=24) + DEF8(=16) = 合計240 TP → TP上限を大幅に超過するためエラーになります。ステータスを下げるか、レベルを上げて調整してください。

> 💡 **ヒント**: クリエイターズ工房の「**バランス計算**」タブで、ステータスを入力するとTP消費が自動計算されます。テンプレートを書く前にバランスを確認しておくと効率的です。

### 5.2 アイテムテンプレート

```json
{
  "$template": "wirth-dawn-ugc",
  "version": "1.0",
  "type": "item",
  "item": {
    "name": "守護獣の牙",
    "type": "trade_good",
    "description": "鎮められた守護獣から得た牙。",
    "image_url": "ugc://images/items/fang.webp"
  }
}
```

* **作成可能なタイプ**: `consumable`（消耗品）、`trade_good`（換金素材）のみ
* **作成不可**: 武器・防具・装飾品・重要アイテムは作成できません
* 消耗品で設定できる効果は `HP回復`（最大200）と `ステータス異常解除` のみです
* 生命力（VIT）を回復するアイテムは作成できません
* すべてのカスタムアイテムの売却価格は **1G固定** です

### 5.3 スキルカードテンプレート

```json
{
  "$template": "wirth-dawn-ugc",
  "version": "1.0",
  "type": "skill_card",
  "card": {
    "name": "ガーディアンストライク",
    "power": 12,
    "ap_cost": 3,
    "target_type": "single_enemy",
    "effect_id": "attack",
    "description": "守護獣の力を宿した一撃"
  }
}
```

* **威力上限**: `power` は最大 **25** です
* **APコスト**: 1〜5の範囲で設定してください
* **使用可能な効果**: `attack`（攻撃）、`pierce_attack`（貫通）、`multi_attack`（連撃）、`heal`（回復）、`buff_self`（自己強化）、`buff_party`（味方強化）、`debuff_enemy`（弱体化）、`aoe_attack`（全体攻撃）
* **使用不可**: 即死効果、反動攻撃

### 5.4 NPCテンプレート

```json
{
  "$template": "wirth-dawn-ugc",
  "version": "1.0",
  "type": "npc",
  "npc": {
    "name": "森の狩人マリア",
    "level": 12,
    "atk": 8,
    "def": 5,
    "durability": 100,
    "cover_rate": 15,
    "ai_role": "striker",
    "signature_skills": ["arrow", "double_slash"],
    "flavor_text": "森に詳しい女性の狩人。"
  }
}
```

* `ai_role`: `striker`（攻撃重視）、`guardian`（防御重視）、`medic`（回復重視）
* NPCの行動は自動（AI制御）です

**NP（NPCポイント）制限**: エネミーのTP制と同様に、NPCのステータスにも「NP制」の制限があります。NPCのレベルに応じたNP上限内にステータスを収める必要があります。

| NP上限の計算 | 10 + レベル × 5 |
|-------------|----------------|
| Lv5のNPC | NP上限 35 |
| Lv12のNPC | NP上限 70 |

| ステータス | NPコスト |
|-----------|--------|
| ATK +1 | 2 NP |
| DEF +1 | 2 NP |
| 耐久度 +10 | 1 NP |
| カバー率 +5% | 2 NP |
| スキル1つ | 3 NP |

例: Lv12のNPC（NP上限70）で ATK8(=16) + DEF5(=10) + 耐久度100(=10) + カバー率15%(=6) + スキル2つ(=6) = 合計48 NP → 上限内なのでOKです。

> 💡 **ヒント**: クリエイターズ工房の「**バランス計算**」タブの「NPCモード」でNP消費を事前に確認できます。

## 7. 報酬のルール — パワーバジェット制

### 7.1 パワーバジェットとは
クエスト報酬としてカスタムアイテムやスキルを配置できますが、**パワーバジェット（PB）** の範囲内に収める必要があります。これは、簡単なクエストで強力な報酬が得られないようにするためのバランス調整システムです。

### 7.2 パワーバジェットの計算

あなたのクエストのパワーバジェットは、以下の式で自動計算されます。

```
パワーバジェット = 推奨レベル × 2 + バトル回数 × 5 + 総ノード数 × 1
```

| クエスト例 | 計算 | PB |
|-----------|------|-----|
| Lv5、バトル1回、全4ノード | 5×2 + 1×5 + 4 | **19** |
| Lv10、バトル2回、全8ノード | 10×2 + 2×5 + 8 | **38** |
| Lv30、バトル3回、全15ノード | 30×2 + 3×5 + 15 | **90** |

→ バトルが多く、長いクエストほど、より良い報酬を設定できます。

### 7.3 報酬のパワーコスト

報酬にはそれぞれ「パワーコスト（PC）」があり、全報酬の合計がパワーバジェット以内であれば設定可能です。

| 報酬種別 | パワーコスト |
|---------|------------|
| HP回復アイテム | 回復量 ÷ 10（例: HP80回復 → 8PC） |
| ステータス回復アイテム | 5PC |
| 換金素材 | 3PC |
| スキルカード | 威力に応じて変動（詳細はバリデーション結果で確認できます） |

**合計PC ≤ パワーバジェット** であればインポート成功です。超過すると、推奨レベルを上げるかバトルを追加してバジェットを増やすか、報酬を減らすよう求められます。

> 💡 **ヒント**: クリエイターズ工房の「**バランス計算**」タブの「クエスト報酬モード」で、推奨レベル・バトル数・ノード数・報酬を入力すると、パワーバジェットの消費が自動計算されます。テンプレートを書く前にバランスを確認しておくことをお勧めします。

### 7.4 設定できない報酬

以下の報酬はゲームバランス維持のため設定できません。

| 禁止報酬 | 理由 |
|---------|------|
| ゴールド | 経済バランスの保護（クリア時に50G固定支給） |
| 経験値 | レベリング速度の公式管理（クリア時に30EXP固定支給） |
| 名声 | ゲーム進行条件の公式管理 |
| アライメント変動 | 世界情勢への影響の公式管理 |
| VIT回復アイテム | 寿命システムの根幹を守るため |
| 武器・防具・装飾品 | 戦闘バランスの保護 |

## 8. 公開までの流れ

### 8.1 テストプレイ
* 公開申請の前に、制作したクエストを自分自身でプレイしてクリアする必要があります。
* テストプレイはペナルティなしの安全な環境で行えます（VIT消費なし、報酬なし）。
* テストプレイをクリアすると「テスト済み」マークが付きます。
* テンプレートを修正して再インポートした場合、テスト済みマークはリセットされ、再度テストプレイが必要です。

### 8.2 審査
* 公開申請されたクエストは、運営チームが内容を確認します。
* 審査期間中、クエストのステータスは「審査中」と表示されます。
* 承認されると自動的に掲示板に掲載されます。
* 却下された場合は理由が「マイ作品」画面に表示されます。内容を修正して再申請できます。

### 8.3 公開後の管理
* 公開中のクエストは「マイ作品」タブから状態を確認できます。
* プレイ回数とクリア回数の統計が表示されます。
* 公開中のクエストはいつでも「取り下げ」できます。取り下げるとドラフト状態に戻ります。再公開するには再度テストプレイと審査が必要です。

### 8.4 公開レートリミット
1日に公開申請できる回数には上限があります。

| プラン | 1日あたりの公開申請回数 |
|--------|---------------------|
| Free | 1回 |
| Basic | 3回 |
| Premium | 10回 |

## よくある質問（FAQ）

**Q: MDテンプレートとJSONテンプレートの違いは？**
→ 内容は同じです。MDは日本語ベースで直感的に書ける形式、JSONはプログラマー向けの構造化された形式です。お好みでお選びください。

**Q: カスタム画像やBGMがなくてもクエストを作れますか？**
→ はい。すべてのアセットは任意です。公式の背景画像やBGMを使うか、何も指定しなければデフォルト素材が使用されます。

**Q: UGCクエストで通常クエストと同じ量のゴールドやEXPは稼げますか？**
→ いいえ。UGCクエストの固定報酬は50G・30EXPです。公式クエストと比べると少額ですが、制作者が用意したカスタムアイテムやスキルカードが手に入る魅力があります。

**Q: UGCクエストで手に入ったスキルカードは通常のバトルでも使えますか？**
→ はい。カスタムスキルカードは通常のスキルカードと同じようにデッキに組み込んで使用できます。

**Q: テストプレイ中に負けたらどうなりますか？**
→ テストプレイ中はペナルティが発生しません。何度でもやり直して、クエストが正しくクリアできることを確認してください。

**Q: 公開したクエストを後から修正できますか？**
→ 一度取り下げてドラフト状態に戻してから、テンプレートを修正して再インポートし、テストプレイ→再審査の手順で更新できます。

**Q: 名もなき旅人の拠所以外でUGCクエストを受注できますか？**
→ いいえ。UGCクエストは名もなき旅人の拠所のギルド（クエストボード）からのみ受注できます。

**Q: バランス計算ツールとは何ですか？**
→ クリエイターズ工房の「バランス計算」タブにある補助ツールです。エネミーのTP、NPCのNP、クエスト報酬のパワーバジェットを、ステータスを入力するだけで自動計算できます。テンプレートを書く前にバランスを確認しておくと、バリデーションエラーを未然に防げます。

**Q: レベル5未満ですが、UGCクエストの内容は見れますか？**
→ はい。クエスト一覧や詳細は閲覧できますが、受注はレベル5以上になってからです。

**Q: AIプロンプトガイドとは何ですか？**
→ テンプレートタブの「🤖 AIで作成」ボタンから利用できる機能です。ChatGPT、Gemini、Claude等の対話型AIにコピペするためのプロンプトが自動生成されます。AIとの対話を通じて、フィールド制約やバランスルールに準拠したテンプレートを簡単に作成できます。テキストエディタでの編集が難しいスマートフォンユーザーに特におすすめです。

**Q: スマートフォンでクエストを作るにはどうすればいいですか？**
→ 以下の3つの方法があります。おすすめ順に：
1. **簡易ビルダー**（「簡易作成」タブ）— ステップごとのUI操作でクエストを組み立て。最も簡単。
2. **AIプロンプトガイド**（「テンプレート」タブ → 🤖 AIで作成）— AIとの対話で作成。高いカスタマイズ性。
3. **MDテンプレート手動編集**（「テンプレート」タブ → MD）— テキストエディタで編集。上級者向け。
